\chapter{Parallelization and Heuristic Dispatching Framework} 
\label{chap:ch6}
\hspace{\parindent}
The results presented in the previous chapter showed that no single prioritization heuristic
dominates across all tested scenarios. Instead, different heuristic strategies may produce better
solutions depending on the structure of each planning instance. This observation motivates the
parallel evaluation of multiple heuristic-guided priority orderings, with the objective of improving
solution quality without increasing the planning latency proportionally to the number of evaluated
heuristics.

This chapter presents the parallelization framework developed around the TEA* planner. The
proposed approach executes several independent TEA* instances in parallel, each using a different
priority ordering generated by one of the available heuristics. The resulting candidate plans are
then compared, and the best valid plan is returned to the Supervisor. Section~\ref{sec:parallel_planning_approach}
describes the overall parallel planning approach and the heuristic dispatching mechanism.
Section~\ref{sec:parallel_implementation_details} presents the main implementation details,
including thread management, synchronization, timeout handling, and controlled termination.
Finally, Section~\ref{sec:redundant_heuristics} discusses how redundant heuristics are handled,
particularly in cases where different heuristics frequently generate equivalent priority orderings
but have different computation costs.
