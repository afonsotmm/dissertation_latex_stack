\section{Experimental Protocol}
\label{sec:parallel_experimental_protocol}
\hspace{\parindent}
The experimental protocol compares the sequential random-ordering baseline with the proposed
parallel framework. Both configurations were evaluated on the same three large-scale scenarios to
ensure that the observed differences result from the planning strategy rather than from changes in
the problem instances.

The sequential baseline evaluates random robot priority orderings with TEA* until either a complete
valid solution is found or the execution limit \(T_{\max}=15\,\mathrm{s}\) is reached. This value was
chosen as a practical upper bound, since even a \(15s\) planning interval is already
long for an online multi-robot coordination system. For each scenario, this process is repeated
over 100 independent runs. In each successful run, the time required to
obtain the first valid solution and its corresponding
cost are recorded. Runs that fail to find a complete valid
solution within \(T_{\max}\) are counted separately and excluded from the mean time and cost
calculations, so the reported averages are computed only over successful runs.

The proposed framework was evaluated using \(1\), \(3\), \(5\), and \(6\) worker threads. For each
run, both the initial ordering of heuristic groups in the priority queue and the ordering of
heuristics within each group were randomized, preventing the evaluation from manually favouring any
specific heuristic family for a given scenario. After this initialization step, the dispatching
mechanism follows the group usage counters described in Chapter~\ref{chap:ch7}, prioritizing
heuristics from less used groups throughout the planning cycle.

Each parallel configuration was also executed 100 times per scenario, with each run recording the
time to the first valid solution, the cost of the first valid solution, the time at which the best
solution was found, the cost of the best solution, and the time required to complete the evaluation
of the heuristic pool. In this context, the best-known solution corresponds to the lowest-cost
solution identified for the same scenario in the heuristic evaluation of Chapter~\ref{chap:ch6}.
Therefore, the time to the best solution measures how quickly the parallel framework reaches the
best solution previously obtained by the individual heuristic analysis.

In addition to the mean values reported in the tables, Empirical Cumulative Distribution Functions
(ECDFs) are used to complement the analysis of planning times across the 100 runs. These curves are
reported for the time to the first valid solution and for the time to the best-known solution,
allowing the consistency and convergence behaviour of each configuration to be compared beyond the
average values alone. The sequential baseline is included in these plots only when it provides a
meaningful number of valid runs.

\subsection{Scenario 1}
\label{subsec:parallel_results_scene1}
\hspace{\parindent}
The first large-scale scenario, described in
Section~\ref{subsubsec:large_scale_docking_to_warehouses}, involves movements from docking
areas towards warehouse regions. As shown in Table~\ref{tab:parallel_results_scene1}, the

Section~\ref{subsubsec:large_scale_docking_to_warehouses}, where random priority orderings were
shown to be ineffective for this problem instance. In contrast, all heuristic-based configurations
solved every run, showing that the developed framework substantially improves planning
robustness in this instance.

\section{Overall Performance Discussion}
\label{sec:parallel_overall_performance}
\hspace{\parindent}
The results across the three large-scale scenarios show that the proposed framework improves the
practical behaviour of TEA*-based planning in two main ways. First, the use of heuristic-generated
priority orderings substantially increases robustness when compared with random priority
permutations. This is particularly evident in Tables~\ref{tab:parallel_results_scene1}
and~\ref{tab:parallel_results_scene3}, where the sequential random-ordering baseline either failed
to obtain any valid solution or solved only a very small fraction of the runs within the adopted
execution limit of \(T_{\max}=15\,\mathrm{s}\). In contrast, all heuristic-based configurations obtained valid solutions in every run, which is
consistent with the heuristic evaluation reported in Chapter~\ref{chap:ch6}, where the large-scale
instances were shown to be highly sensitive to the selected robot priority ordering.

Second, parallel execution reduces the time required to explore useful regions of the priority
ordering space. With a single worker, the framework evaluates only one heuristic ordering at a time, avoiding most
of the overhead associated with concurrent execution but making its ability to find a good ordering
early entirely dependent on the first heuristics selected from the pool.
As a result, the single-threaded configuration can occasionally produce an early valid solution, as
visible in the ECDFs of Figures~\ref{fig:scenario1_time_ecdfs}
and~\ref{fig:scenario3_time_ecdfs}, but its curves rise more gradually, indicating lower consistency
across runs.

Increasing the number of worker threads improves the probability of evaluating a useful heuristic
ordering in the first planning batch, thereby explaining the clear reduction in the time to the first
valid solution when moving from one thread to three or five threads, as shown in
Tables~\ref{tab:parallel_results_scene1},~\ref{tab:parallel_results_scene2},
and~\ref{tab:parallel_results_scene3}. The corresponding ECDFs also show that the multi-threaded
configurations generally shift the distributions towards shorter planning times. The five-thread
configuration is particularly effective for early response, obtaining the lowest mean time to the first
valid solution in Scenarios~1 and~3 and the lowest mean first solution cost in Scenarios~1 and~2.
This suggests that evaluating a broader set of heuristic alternatives in parallel is beneficial not only
for feasibility, but also for the quality of the first solution returned by the planner.

However, the gains are not linear with the number of threads, as the six-thread configuration does
not consistently improve the time to the first valid solution over five threads and is slightly slower
in some cases, as shown in Tables~\ref{tab:parallel_results_scene1}
and~\ref{tab:parallel_results_scene3}, as well as in the first valid solution ECDFs. This behaviour is expected because each additional worker introduces
scheduling and synchronization overhead, while each TEA* instance also competes for CPU and
memory resources. Since the experiments were executed on a machine with six physical CPU
cores, using six worker threads leaves little processing headroom for the operating system and for
the remaining execution overhead. Therefore, the additional parallelism can be partially offset by
resource contention in the early-response metrics.

This limitation is strongly dependent on the hardware used for execution. On a machine with a
larger number of physical CPU cores than the number of heuristic evaluations launched in a
planning cycle, the framework would be able to evaluate more, or even all, heuristic alternatives with less resource contention. In that setting, the time required to explore the heuristic
pool would be closer to the duration of the slowest individual TEA* execution rather than to the
sum of several sequential batches. The results obtained in this work should therefore be interpreted
as the performance of the proposed framework under the available computational resources, rather
than as an intrinsic upper bound of the parallelization strategy.

The effect of higher thread counts becomes more favourable when considering the time to the
best-known solution and the time required to complete the heuristic pool. In
Tables~\ref{tab:parallel_results_scene2} and~\ref{tab:parallel_results_scene3}, the six-thread
configuration reaches the best-known solution fastest and also completes the heuristic pool in the
shortest mean time. The ECDFs in Figures~\ref{fig:scenario2_time_ecdfs}
and~\ref{fig:scenario3_time_ecdfs} support the same interpretation, with higher thread counts
reaching large cumulative fractions earlier for the best-known-solution metric. This indicates that
using more workers is advantageous when the objective is not only to obtain a valid solution
quickly, but also to evaluate more heuristic alternatives within a shorter planning interval.

Overall, the results show that the proposed framework improves TEA*-based multi-robot planning
by combining heuristic diversity with parallel execution. The heuristic orderings address the
limitations of random priority sampling in difficult scenarios, while the parallel architecture reduces
the time required to test multiple promising orderings. The observed performance therefore results
from the interaction between the quality of the heuristic priority orderings, the degree of parallelism
used to explore them, and the computational resources available on the execution platform.
